# Site Capture UI 기준

## 목적

업무용 캡처 도구에서 입력, 진행 상태, 결과를 빠르게 읽고 조작할 수 있게 한다.

## 토큰

- 글꼴: A2Z, 사용할 수 없을 때 Malgun Gothic
- 본문: 14px, 제목: 16px, 화면 제목: 22px
- 배경: `#F5F7FA`, 표면: `#FFFFFF`, 본문: `#172033`, 보조 텍스트: `#64748B`
- 강조: `#2563EB`, 실패 행: `#FFE8E8`
- 간격: 4px 단위. 카드 안쪽 20px, 카드 사이 16px

## 레이아웃

- 창 최소 크기: 1180 × 820, 기본 크기: 1500 × 1000
- 실행·결과·설정 탭의 본문은 각 탭이 하나의 세로 스크롤 영역을 가진다.
- 긴 설명, 상태, 완료 세부 정보는 줄바꿈한다.
- 실행 제어와 최근 결과는 1:1 비율로 한 줄에 배치한다.
- 최근 결과 카드는 실행 카드와 같은 높이로 늘어난다. 기본 창 크기에서는 실행 탭의 세로 스크롤이 생기지 않아야 한다.
- 최근 결과의 제목과 설명은 카드 상단에 붙이고, 나머지 영역은 결과 테이블이 전부 채운다.
- 완료·성공·실패 수치와 진행 바는 실행 버튼 바로 아래에 배치한다.
- 실행 버튼은 왼쪽 영역 안에서 2열 그리드로 배치해 좁은 창에서도 한 줄을 넘지 않는다.
- 표의 긴 텍스트는 말줄임표와 툴팁으로 전체 내용을 보완한다.

## 구성 요소

- 카드: 흰 배경, 12px 반경, 1px 테두리
- 주요 버튼: 파란색 채움
- 보조 버튼: 흰 배경 테두리
- 상태 카드: 상태 배지, 세부 설명, 진행률을 세로로 구분
